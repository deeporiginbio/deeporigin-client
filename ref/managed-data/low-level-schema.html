    Low-level response schemas

    The deeporigin.managed_data.schema module contains Pydantic BaseModels that describe responses from Deep Origin's managed data API, and literals that describe possible values for certain queries.

    These models are used both to validate responses and to generate mock data for testing.

    class ColumnItem(BaseModel):
    """Schema for items in the `cols` attribute of responses
    from the `DescribeRow` endpoint for a database."""

    id: str
    name: str = "Placeholder Name"
    key: str
    parentId: str = "db-placeholder-1"
    type: DataType = "text"
    dateCreated: str = "2024-04-04T17:03:33.033115"
    cardinality: Cardinality = "one"
    canCreate: Optional[bool] = False
    configSelect: Optional[dict] = None

    model_config = ConfigDict(extra="forbid")

    class DescribeRowResponse(BaseModel):
    """Schema for responses from the `DescribeRow` endpoint. This schema is complex because

    id: str
    hid: str

    parentId: str

    dateUpdated: str = "2024-04-04 16:33:58.622469"
    createdByUserDrn: str = "placeholder"
    rowJsonSchema: dict = {"type": "object", "required": [], "properties": {}}
